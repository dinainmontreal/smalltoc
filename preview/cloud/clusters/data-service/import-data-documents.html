Import Data

      +
      Import CSV and TSV files, JSON docs, or JSON Lines to your Capella database from your local machine.

      You can choose to import datasets smaller than 40 MB directly from your browser. For larger datasets, you can choose to import using the command line. You can import sample data to explore and learn more about Capella.

      Capella supports importing these file formats:

      • CSV - Comma-separated value, map the CSV columns to fields.

      • TSV - Tab-separated value, each line represents a row of the table, and individual data fields are separated by tabs

      • JSON - JavaScript Object Notation documents

      • JSONL - JSON Lines format, a structured data storage format where each line is treated as an independent, valid JSON object

      You can import your dataset to Capella from a local file.

      Each document extracted from your imported files needs a document key. You can choose to let Capella generate keys randomly for each document or specify a field value from your dataset for the document key. You can also choose to add a custom expression to generate document keys.

      Access Import in the Capella UI

      To access the Import tool, go to Data Tools  Import.

      In trial accounts, the Start tab also has a link to the Import tool.

      Import Data from Your Browser

      Import a single file smaller than 40MB.

      1. Go to Data Tools  Import.

      2. Choose Load from your browser.

      3. Click Upload to choose the JSON, TSV, or CSV file to import, or drag-and-drop it into the UI.

      4. Use the Bucket, Scope, and Collection lists to choose where in your database you want to import your file.

      5. (Optional) To add a new bucket, scope, or collection for your data, click +Create.

        1. Choose New and enter a name for your bucket, scope, or collection, and click Create.

          For more information about how to create a bucket with custom settings, see Create a Bucket.

      6. Choose how to create your document key values:

        UUID

        This is the default option that generates a random Universally Unique Identifier for each document.

        Field

        To generate the document key from a field name value, choose a field name from the list.

        Custom

        To generate custom keys, you can use a combination of the following:

        • #UUID# for UUIDs

        • #MONO_INCR# for monotonically increasing numbers

        • % % for a field value from a document, for example %insurance_number%. Separate nested field names with dots, for example key::%name.full.name%.

        • Any other separators, punctuation, or text

      7. Check that your chosen document keys are correct:

        1. Click DocIDs and Documents.

        2. In the document preview, check that the "docID" value for the documents in your uploaded file displays as expected.

          The preview indicates the numbers of documents in your source file. You can also view your data as a raw file or a parsed table.

      8. Click Import Rules.

      9. Configure which parts of the source you want to import in the Filter Imported Data window, and click Save.

        • JSON/JSONL

        • CSV/TSV

        • Skip the first n documents: by default, all documents are loaded. Setting this option skips a specified number of documents before importing. When enabled, enter the number of documents to skip before the import starts.

        • Import a maximum of n documents: stop loading after a specified number of documents. Use this to partially load large datasets. When enabled, enter the maximum number of documents to import.

        • Exclude field names: Omit named fields from the uploaded documents. Use this to remove a field from documents, for example, a document key from a field in a document. When enabled, specify a comma-separated list of field names to exclude from the import.

        • Skip the first n documents: by default, Capella loads all documents. Setting this option skips a specified number of documents before importing. When enabled, enter the number of documents to skip before the import starts.

        • Import a maximum of n documents: stops loading after a specified number of documents. Use this to partially load large datasets. When enabled, enter the maximum number of documents to import.

        • Exclude field names: Omit named fields from the uploaded documents. Use this to remove a field from documents, for example, a document key from a field in a document. When enabled, specify a comma-separated list of field names to exclude from the import.

        • Omit Empty Types: rows in the CSV file which don’t contain data are stored as empty strings in the JSON doc. Setting this option removes these fields from the uploaded document.

        • Infer Field Types: all values in a CSV are interpreted as strings by default. Setting this option causes the Import tool to look at each value and decide if it’s a string, integer, or Boolean value and put the inferred field type into the document.

      10. Click Import.

        When the import is complete, a banner appears at the top of the page to tell you it was successful.

        To view your documents, click the Documents tab.

      Import Data Using cbimport

      1. Go to Data Tools  Import.

      2. Choose Load with cbimport.

      3. Click Upload to upload a sample file from your dataset, or drag-and-drop it into the UI.

      4. Use the Bucket, Scope, and Collection lists to choose where in your database you want to import your file.

      5. (Optional) To add a new bucket, scope, or collection for your data, click +Create.

      Import Sample Data

      Load sample data to try out Capella’s Data Tools.

      1. Go to Data Tools  Import.

      2. Choose Load sample data.

      3. Choose a sample.

      4. Click Import.

      Next Steps

      The Import tool loads data into a bucket on your database. If you wish to restore a bucket of data, refer to the Restore a Bucket page.
